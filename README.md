# Indikin Ecosystem Draft Documentation

Welcome to the central documentation and strategic planning repository for the Indikin ecosystem. This draft suite serves as the canonical source of truth for developers, stakeholders, and AI agents contributing to Indikin.

## Overview

This repository is designed specifically for **AI-ready ingestion** and structured team collaboration. It houses three core domains of knowledge: the technical foundation, the business strategy, and the operational AI budget.

## Repository Structure

### `📁 .brand/` (Engineering & Design Canonical)
The definitive guide for building and maintaining the Indikin platform. It contains structured, AI-ready references for code generation:
- **[BRAND_IDENTITY.md](.brand/BRAND_IDENTITY.md)**: Mission, vision, core assets, typography, color palettes, spacing, and legal copy.
- **[DESIGN_SYSTEM.md](.brand/DESIGN_SYSTEM.md)**: CSS tokens, Tailwind aliases, button variants, and comprehensive component specifications.
- **[ARCHITECTURE.md](.brand/ARCHITECTURE.md)**: System topology, Firestore schemas, PulseChain/Ecosystem integrations, security models, and deployment configurations.
- **[API_REFERENCE.md](.brand/API_REFERENCE.md)**: Detailed API endpoint documentation, Cloudflare Worker architecture, and authentication models.

### `📁 .brain/` (Strategy & Business Intelligence)
The core business logic, pitch decks, and market strategy for Indikin. Includes:
- **Tokenomics**: Briefs, conservative models, and ecosystem integration patterns.
- **Go-To-Market (GTM) Strategies**: Dedicated playbooks for Filmmakers, Fans, and Liquidity Providers.
- **Business Operations**: Pitch decks (Ethereum Seed, Creator/Fan decks), business plans, financial projections, roadmap 2026, competitive breakdowns, and team CVs.

### `📁 .budget/` (AI Operations & Planning)
A focused workspace dedicated to detailing and modeling Indikin's AI implementation costs:
- **Assumptions & Hardware**: Hardware considerations, staffing, tooling, and core assumptions.
- **Modeling**: Base/upside/downside scenario modeling and line-item budget drafts.

## How to Use This Repository

For AI agents and human contributors alike:
1. **Technical Ingestion**: Start by reading `.brand/README.md`. It provides a detailed index of where to look for routing, API calls, design variables, and global context.
2. **Business Alignment**: When drafting copy, designing campaigns, or considering token-related features, refer to the deep strategic files in `.brain/`.
3. **Budgetary Constraints**: Always abide by the structural limitations and calculations established in `.budget/`.
